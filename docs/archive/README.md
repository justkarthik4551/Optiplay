# Archive — Process Documentation

> **OptiPlay — Gamified Operations Research Learning Platform**

---

This folder contains the **process documents** created during the build lifecycle of OptiPlay. They document the *journey* — strategy, product decisions, sprint planning, and risk management.

For the **outcome** documentation (what the project *is* and *how it works*), see the numbered root docs in [`/docs/`](../README.md).

---

## Contents

| Folder | Documents | Purpose |
|--------|-----------|---------|
| `01_strategy/` | Project Charter, Competitive Analysis | Why and what we built |
| `02_product/` | PRD, Design System, Personas, User Stories, Wireframes | Product requirements and UX design decisions |
| `03_engineering/` | System Architecture (v1), Algorithms & Logic (v1), GDDs | Technical specs — **superseded** by numbered root docs |
| `04_delivery/` | Dev Plan, Sprint Backlog, Risk Register, Timeline, Deployment Guide, Walkthrough | Build management and delivery records |

---

> **Note on `03_engineering/`:** These documents were the original technical specs. They have been superseded by the more comprehensive, cross-linked numbered docs in `docs/`. They are preserved here for full audit trail, but may contain outdated information (e.g., they do not cover Game 2 "Pack the Truck").
